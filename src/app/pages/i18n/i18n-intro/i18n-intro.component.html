I18N (Internationalization) and Localization

We have two different options available for translations. We recommend reading the documentation for each to determine which is best for your needs.

  • Angular I18N. This is the translation paradigm built into Angular. It requires creation of different application instances for each language. These instances are created at build time.
  • ngx-translate. This 3rd party library allows string replacement at run time.
