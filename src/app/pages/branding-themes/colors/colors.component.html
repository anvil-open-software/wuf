KION Color System

The KION Color System is an organized structure for defining a set of colors to use throughout the UI.

Semantic Naming Convention

The KION Group is comprised of a number of different brand identities, each with their own distinct theme elements. The KION Color System is designed to accommodate the unique requirements and characteristics of these individual brands while still remaining constrained by an organized system.

In order to provide the necessary structure, the complete color system, and the colors within it, will be identified by a semantic naming convention based on usage, context, and state and not their values. When these colors are applied in the application code, it is important that the semantic naming system be used when referencing the color system and in lieu of the individual colors or color values that ultimately appear in the UI.

Every color in the color palette has a corresponding CSS custom property name for use in the code. Each CSS custom property name is namespaced with the prefix "--wuf-" to help avoid name collisions with third party CSS custom property names (like those from Material Design itself). The color swatches for each color shown below all include their corresponding CSS custom property names.

Base Colors

In the KION Color System, a primary, secondary, and accent color (note the use of the semantic naming convention) are chosen to represent a brand. Lighter and darker variants of the three based colors can then be applied to the UI in different ways (see "Color Variations" below).

primary
--wuf-color-primary

The primary color is typically the brightest and most easily identifiable color within a brand's identity. If only one color must be selected to represent a brand's entire identity, it is the primary color.

secondary
--wuf-color-secondary

The secondary color plays a support role and is typically the more subdued color of a brand's identity. While it may be more subdued, it is often/usually the color that constitutes the greatest area of a UI in terms of pixel count. Application framing and background colors often use the secondary color.

accent
--wuf-color-accent

The accent color is used for floating action buttons and other interactive elements, serving as visual contrast to the primary. UI elements that must present a selected state, active items, and enabled states use the accent color for this purpose.

State Colors

In addition to the three base colors, four additional colors are used to represent state. State colors are comprised of success, info, warning, and danger.

State colors can and should remain within the same color family as the three base colors and/or be complimentary to those colors. State colors are also allowed to reuse the three base colors. However, it is important that state color choices should allow those colors to be easily differentiable from the three base colors within the UI based on context and usage. If it is not possible to easily and consistently present a state color as apart from the surrounding brand colors, it is recommended to select state colors that avoid ambiguity altogether. For example, the Linde primary color is a rich red. If a rich red is the dominant color within the UI for branding purposes, the same red cannot also be used to denote an error state (i.e., the "danger" color). This is because re-purposing a red for both branding and error states would not allow the error state color to call sufficient attention to itself. When choosing colors for the KION Color System, therefore, either the primary or the error color can use red, but not both.

NOTE: Material Design includes several additional state colors including disabled and hover, most of which indicate state based on user interaction. Since most of these additional colors are included for us algorithmically, there's no need to explicitly call them out in this guide. For more information on state visit https://material.io/design/interaction/states.html.

success
--wuf-color-success

The success color is used to convey successful completion of an user activity to call attention to a positive metric or condition. Given that positive metrics or conditions (hopefully the default state), it's generally not necessary to call attention to UI elements using success coloring so this color is generally reserved for success messaging following user activity.

info
--wuf-color-info

The info color is used to convey information of an informative, non-urgent nature. Users are not required to take action when informational data is displayed.

warning
--wuf-color-warning

The warning color is used to convey information of a potentially serious nature that the user may need to act on. The state is not yet non-blocking but may become so unless the user acts.

danger
--wuf-color-danger

The danger color is used to indicate a blocking or error condition that the user must act on before they can proceed. A system in this state is blocking in nature and requires immediate attention and action.

For more information on state usage with Material Design, visit https://material.io/design/interaction/states.html
