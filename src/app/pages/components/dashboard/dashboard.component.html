API

The WufDashboardModule is comprised of two different components working together:

  • The <wuf-dashboard> component, which handles the overall layout of the dashboard.
  • Any number of <wuf-kpi> components, which make up the contents of the dashboard and display corresponding Key Performance Indicators on the left side of the layout.

Both of these components are included as a part of the WufDashboardModule module from the @anviltech/wuf-ang-dashboard package in Artifactory.

<wuf-kpi> Usage

Now that we have the dashboard framework in place, we can add some content to it. Dashboard content comes in the form of <wuf-kpi> components. To add content to a dashboard, place as many <wuf-kpi> tags inside the <wuf-dashboard> tag as you'd like:

The only required attributes for the <wuf-kpi> component are label and value. When added, a new KPI will appear in the KPI section on the left displaying the label and value provided. Additionally, the content you provided within the <wuf-kpi> tags will be shown on the right.

If you have added multiple <wuf-kpi> tags to the parent <wuf-dashboard> tag, the <wuf-dashboard> tag will ensure that only one KPI will be active at a time and that only the contents of that active KPI will be displayed in the content area. KPIs behave like tabs in this way and display their corresponding content only when clicked and active. The first KPI added to the page is the initially active KPI on page load.

Dyname Value Sizing

Notice that in the example to the left the "API" section displays a long string of 9 characters. In order for the entire string to be visible, the <wuf-kpi> tag is dynamically sizing the value element based on string length. Additionally, the text size of the value will grow/shrink as the available realestate within the browser grows/shrinks. It does this using a combination of SVG, CSS, and some javascript. This mechanism ensures that KPI values can be displayed on as many different devices as possible, including mobile devices where space is severely limited. The value can handle about 13 characters before it starts getting truncated.

<wuf-dashboard> Properties

Property Type Default Description
minKpiCount number 4 Minimum number of KPIs to show. If the dashboard includes fewer than this number of <wuf-kpi> elements, filler KPIs will be added to match this number of displayed KPIs. Set to 0 to allow any number of KPIs.

<wuf-kpi> Properties

Property Type Default Description
color string None If added, this optional value will change the color of the right border. Any color name from the color palette (primary, secondary, accent, success, info, warning, danger) can be used here. This is a good way to indicate a KPI is in a state that requires user attention.
label string None The label to display in the KPI. Long strings will wrap.
padding string 20px The amount of padding to display around content in the content area. Sometimes you may want to remove all padding in order to insert a split panel that doesn't need any padding. In this case you can pass any CSS size value here (e.g., px, em, rem, pt, etc.)
value string|number None (required) The value to display in the KPI. Values longer than about 13 characters in length will be cropped, depending upon available screen size and number of KPIs displayed.

CSS Custom Properties

CSS Custom property Description Value
--wuf-dashboard-content-padding The CSS value for padding to use in the content area of the dashboard. 20px
--kpi-border-color The color for the line vertically separating the KPIs. var(--wuf-kpi-border-color, var(--wuf-color-gray-400))
--kpi-stripe-color The color of the thick vertical line appearing on the right side of the KPI. var(--wuf-kpi-stripe-color, var(--kpi-border-color))
--kpi-label-color The color of the label appearing above the KPI value. var(--wuf-kpi-label-color, var(--wuf-color-gray-600))
--kpi-bg-color The background color of the KPI element. var(--wuf-kpi-bg-color, var(--wuf-color-gray-200))
--kpi-active-bg-color The background color of the KPI element in an active state. var(--wuf-kpi-active-bg-color, var(--wuf-color-primary-darker-5))
--kpi-active-stripe-color The stripe color to use for the KPI element in an active state. This color is overridden when the color property is used. var(--wuf-kpi-active-stripe-color, var(--wuf-color-primary-lighter-10))
