The Smart Table component is forked from ng2-smart-table and most of the ng2-smart-table documentation applies to this component. Refer to this documentation first, then read the following for instructions on additions and changes from the source.

Table Configuration Additions

Property Type Default Description
Other Properties

actions.add.position string header The position at which to display the button. If set to "header", the button will appear in the table header, to the right of the table's title (if it exists). Otherwise, the button will appear in the actions column of the filter row (if visible).
actions.add.tip string 'Add New' The text to display in the tooltip when the user mouses over the button/icon. Note that if add.label is overridden to include a string to display as a button label, an icon is not displayed and this property will do nothing. hover boolean false Whether or not to display a highlight color on the row when the user's mouse passes over the row.
noDataMessage string 'No data found' The text to display when no data is loaded into the table.
title string none If a title is included, a title bar will appear in the header to show this string. If this string corresponds with a translation key, ngx-translate will translate it.
valign string 'middle' Sets the vertical alignment of every cell in the table. Possible values are top, middle, bottom.
width string '100%' Takes any valid CSS values for the width property. By default the table will stetch to 100% width of its parent container. By setting the value to "auto" or an empty string ("") the table width will be determined by the contents. Note that if the contents of the table or the number of columns requires the table to stretch wider than 100% of available width of the containing element, the table will be as wide as it needs to be regardless of the value of this setting.

CSS Variables (i.e., CSS Custom Properties) Reference

The following theme-related CSS custom properties are available to use throughout your application's CSS files should you decide to use them to further customize your brand's appearance. By default, the values of these properties are set to use the default brand theme. However, you can easily override all of the following default value with your own theme values using the SCSS methods defined in the Theme System documentation.
