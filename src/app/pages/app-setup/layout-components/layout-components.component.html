Layout Components

The Common Components Library provides a number of components related to layout. These components are intended to handle basic page layouts and orientations.

Page Scaffolding

Basic page scaffolding is provided by the <kg-content-header>, <kg-content-main>, and <kg-content-footer> components.

Every page in your application should include these three components. Even if you don't intend to use all three, we recommend including each for consistency and for future functionality. Consider them layout "slots" into which your page content can be added. The components will handle the generation and styling of your page's layout HTML/CSS while you, as a developer, focus on the page content placed inside each.

<kg-content-header>

The <kg-content-header> slot is used to add a toolbar, breadcrumb trail, and page-level utility links to an application page. Or other content that is not scrollable and need to stay on top.

<kg-content-main>

The <kg-content-main> slot is used to host the main page content. It generates the HTML/CSS necessary to control content padding, content scrolling, etc.

<kg-content-footer>

The <kg-content-footer> slot is used to (optionally) display page-level footer content. A page footer is fixed in position and always appears at the bottom of the content area overlaid on TOP of page content.
